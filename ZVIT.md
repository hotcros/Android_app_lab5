Дослідити яким чином платформа Андроїд надає можливість оброблювати аудіо-файли та відео-файли та отримати практичні навички щодо використання інструментів відтворення медіа-даних.

4. Надати вибір завантаження файлу з Інтернету.
5. Використовувати для реалізації обробки медіа-даних спеціалізовані інструменти (`MediaPlayer`, `VideoView`).

## Відповіді на контрольні питання

**1. Наведіть способи підключення Інтернет ресурсів до мобільного застосунку.**
Для підключення інтернет-ресурсів необхідно додати дозвіл `<uses-permission android:name="android.permission.INTERNET"/>` у маніфесті. Способи підключення включають: виконання HTTP-запитів через бібліотеки (Retrofit, OkHttp) або нативні класи (`HttpURLConnection`); завантаження медіаданих через URL за допомогою медіаплеєрів (`MediaPlayer`, `VideoView`, `ExoPlayer`); використання `WebView` для відображення веб-сторінок всередині застосунку.

**2. Поясніть різницю між внутрішнім та зовнішнім сховищем.**
Внутрішнє сховище (Internal Storage) використовується для зберігання приватних даних додатку. Доступ до них має лише сам додаток, і ці файли видаляються при видаленні застосунку. Зовнішнє сховище (External Storage) призначене для публічних файлів (завантаження, музика, фото), які доступні для перегляду та редагування іншим додаткам і користувачеві.

**3. Наведіть категорії файлів при збереженні в зовнішньому сховищі.**
При збереженні файлів у зовнішнє сховище система використовує стандартизовані публічні каталоги за типом контенту:
* `Music` (аудіо та музика)
* `Movies` (відеофайли)
* `Pictures` (зображення та фото)
* `Downloads` (завантажені користувачем файли)
* `Ringtones`, `Alarms`, `Notifications` (системні звуки)
* `Documents` (текстові файли, PDF тощо)

**4. Опишіть властивості спеціалізованих інструментів для відтворення аудіо-файлів.**
Основним інструментом є клас `MediaPlayer`. Його властивості:
* Підтримка відтворення як з локальних джерел (папка `raw`, пам'ять пристрою), так і з мережевих потоків (URL).
* Наявність вбудованої машини станів (методи `prepare()`, `start()`, `pause()`, `stop()`, `release()`).
* Підтримка асинхронної підготовки (`prepareAsync()`) для уникнення блокування головного потоку при завантаженні з мережі.
* Можливість прослуховування подій (наприклад, `OnCompletionListener` для відслідковування кінця треку).

**5. Опишіть властивості спеціалізованих інструментів для відтворення відео-файлів.**
Основним базовим інструментом є `VideoView` (або більш сучасний `ExoPlayer`). Властивості `VideoView`:
* Інкапсулює в собі `MediaPlayer` та `SurfaceView`, автоматично керуючи відображенням кадрів на екран.
* Дозволяє відтворювати відео як з локальних URI, так і з інтернету (метод `setVideoURI()`).
* Має зручну інтеграцію з `MediaController` для автоматичного додавання візуальної панелі керування (кнопки паузи, перемотки, смуга прогресу).
* Потребує обережного керування життєвим циклом (особливо при повороті екрану або згортанні додатку) для збереження позиції відтворення.
